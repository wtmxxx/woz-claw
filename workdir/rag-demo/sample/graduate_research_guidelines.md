# 研究生科研与论文管理说明

为规范研究生阶段科研训练、论文写作和过程管理，学院制定以下说明。本文件用于示例校园场景中的长文档分片效果，涵盖政策、流程、表格、清单和例外情况。

## 一、总体要求

研究生培养强调科研能力、学术规范与实践能力并重。学生在培养周期内应完成课程学习、文献阅读、开题报告、中期检查、论文写作、学术交流与毕业答辩等环节。不同专业方向可根据培养方案进行适度调整，但以下原则须保持一致：

1. 所有科研活动应符合学术诚信要求，不得伪造数据、篡改结果或重复投稿。
2. 所有实验记录应尽量保留原始文件、代码版本与数据来源说明。
3. 所有论文写作应注明参考文献来源，引用格式保持统一。
4. 涉及人类受试者、动物实验或敏感数据的研究，应按学校伦理审查要求执行。

## 二、阶段安排

### 1. 第一阶段：选题与文献综述

选题阶段的目标不是“尽快定题”，而是尽快形成可解释、可执行、可评估的研究问题。学生应围绕导师方向、个人基础和时间安排综合考虑，避免选题过大、过空或缺乏数据支持。文献综述至少应覆盖以下三个层面：

- 领域背景：研究主题的历史脉络与应用场景。
- 方法演进：已有方法的技术路线、优点与局限。
- 问题缺口：当前研究尚未解决的关键难题。

常见问题包括：文献搜集过多但没有分类，综述只写“别人做了什么”而没有说明“为什么这么做”，以及将课堂资料直接拼接成论文背景。建议在这一阶段建立文献表，字段至少包括作者、年份、研究对象、方法、结论、局限与可借鉴点。

### 2. 第二阶段：开题与方案论证

开题报告应明确研究问题、技术路线、数据来源、实验方案、预期成果和时间计划。学校通常要求开题阶段完成一次组内汇报，并根据反馈修订研究计划。若研究涉及模型训练、问卷调查或校外合作，应在开题中补充可行性说明。

开题汇报时建议重点回答以下问题：

- 为什么这个问题值得研究？
- 现有方法为什么不够好？
- 你的方案相较于已有方案改进在哪里？
- 你如何验证结果有效？
- 如果中途失败，有没有备选方案？

### 3. 第三阶段：中期检查

中期检查主要考察进度，而不是结果是否“已经做完”。学生应提交阶段性成果，包括实验记录、阶段总结、论文初稿或系统原型。学院通常会检查研究是否偏离原题、是否存在明显停滞、是否有充分的时间完成后续工作。

建议中期检查前准备如下材料：

| 材料名称 | 内容要求                       | 提交形式    |
| -------- | ------------------------------ | ----------- |
| 阶段总结 | 已完成工作、遇到问题、后续安排 | PDF         |
| 实验记录 | 数据集、实验配置、结果截图     | 附件        |
| 论文提纲 | 章节结构、每章核心内容         | Word 或 PDF |
| 导师意见 | 导师确认签字或线上批注         | 扫描件      |

### 4. 第四阶段：论文修改与定稿

论文写作过程中最常见的问题不是“不会写”，而是“写完后没有回头统一修订”。建议定稿前至少完成三轮检查：

- 结构检查：章节逻辑是否连贯，是否存在前后矛盾。
- 语言检查：句子是否重复，术语是否统一。
- 格式检查：目录、图表编号、参考文献、页眉页脚是否符合规范。

如果论文中包含表格或图像，应确保其在黑白打印下依然可辨识。若图表占比过大，正文应当解释图表结论，而不是仅仅“放图不分析”。

## 三、学术规范与风险控制

### 1. 数据管理

学生应对原始数据、清洗数据、训练数据和结果文件进行分层管理，建议目录结构如下：

- raw_data：原始数据，禁止直接覆盖。
- processed_data：清洗后的数据，可复现。
- experiments：每次实验的配置与日志。
- drafts：论文草稿与修改版本。

数据管理的核心目的不是让目录“看起来整齐”，而是保证任何一项结果都能追溯到输入来源与处理过程。对于随机性较强的实验，应保存随机种子、参数配置和运行环境。

### 2. 论文署名

若论文涉及多人合作，应在研究初期明确贡献分工，包括问题定义、数据处理、实验设计、模型实现、论文写作与答辩准备。对于共同完成的工作，不得在没有实际贡献的情况下署名。

### 3. 查重与引用

引用别人的观点、定义、流程图或数据集时，应在正文和参考文献中双重标明。查重不是目的，避免抄袭才是目的。很多同学将“改几个词”理解为改写，实际上这仍然可能构成不当引用。最稳妥的方式是：先理解，再用自己的语言重写，并明确指出来源。

## 四、常见情形处理

### 情形一：研究进度落后

如果因为课程任务、实习或身体原因导致进度落后，应尽快向导师说明，不要等到检查前才补材料。建议处理方式如下：

1. 重新拆解任务，判断哪些内容能在两周内完成。
2. 将“理想方案”改成“可交付方案”。
3. 先补充最关键的实验或案例，再完善文字表达。
4. 必要时降低创新目标，优先保证毕业可行性。

### 情形二：实验结果不稳定

实验结果不稳定时，不要反复切换方向。优先排查数据、参数、随机种子、样本划分和环境差异。若结果仍然波动，应在论文中如实说明，并分析可能原因，而不是只展示最好的一次结果。

### 情形三：导师意见变化较大

导师意见变化并不意味着前期工作无效，而是说明研究还在收敛。建议学生保留每次修改记录，形成可追踪的版本历史。对于争议较大的问题，可准备两个方案，以便在汇报时快速切换。
